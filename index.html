AdIntel MCP

Advertiser intelligence for AI assistants — powered by Sensor Tower data, hosted as a read-only MCP server.

https://adintel-delta.vercel.app/
ℹ️ The MCP is read-only. It queries data already stored in the shared database. No data collection runs through this endpoint.

Installation

Claude

1 Claude.ai Chat & Cowork (Web)

No config files needed — use the Connectors UI. Works in both Chat and Cowork tabs.

  1. Open claude.ai
  2. Go to Profile → Settings → Connectors
  3. Click "+" → "Add custom connector"
  4. Enter the URL below and click Add
https://adintel-delta.vercel.app/

To enable in a conversation: Click "+" at the bottom of the chat → Connectors → toggle AdIntel on.

2 Claude Desktop

Edit your config file. Claude Desktop requires the mcp-remote wrapper — it cannot connect to HTTP servers directly.

Config file location

# macOS
~/Library/Application Support/Claude/claude_desktop_config.json

# Windows
%APPDATA%\Claude\claude_desktop_config.json

Add to your config

{
  "mcpServers": {
    "adintel": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://adintel-delta.vercel.app/",
        "--transport",
        "http-only"
      ]
    }
  }
}

Save the file and fully restart Claude Desktop.

3 Claude Code (CLI)

One command to register the MCP server. Direct HTTP — no wrapper needed.

Add the server

claude mcp add --transport http adintel https://adintel-delta.vercel.app/

Verify it was added

claude mcp list

AdIntel tools are available in all Claude Code sessions immediately.

Codex

Codex

Two ways to add AdIntel — CLI is the quickest.

Option 1 — CLI (quickest)

codex mcp add adintel --url https://adintel-delta.vercel.app/

Option 2 — Config file

Edit (or create) ~/.codex/config.toml:

[mcp_servers.adintel]
url = "https://adintel-delta.vercel.app/"

Restart Codex after either method to reload MCP server definitions.

Antigravity

Antigravity

Antigravity stores MCP server config in a JSON file. You can edit it through the UI or directly on disk.

Step 1 — Open the config file

Via the UI:

  1. Launch Antigravity
  2. Click the Agent icon in the Activity Bar (left sidebar)
  3. In the Agent panel, click the ⚙️ gear icon → Manage MCP Servers
  4. Click Edit configuration — this opens mcp_config.json in the editor

On disk directly:

# macOS / Linux
~/.gemini/antigravity/mcp_config.json

# Windows
%USERPROFILE%\.gemini\antigravity\mcp_config.json

Step 2 — Add the server config

Option A — Native HTTP (preferred, no extra dependencies):

{
  "mcpServers": {
    "adintel": {
      "serverUrl": "https://adintel-delta.vercel.app/"
    }
  }
}

Option B — Via mcp-remote bridge (if native HTTP is not supported in your version):

{
  "mcpServers": {
    "adintel": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://adintel-delta.vercel.app/"
      ]
    }
  }
}

Step 3 — Activate

  1. Save mcp_config.json
  2. In the Agent panel, click the ↺ refresh icon, or fully restart Antigravity
  3. The adintel server should appear in the MCP server list with a green indicator

Step 4 — Verify

Ask Antigravity: "What tools do you have access to?"
You should see all AdIntel tools listed — list_advertisers, get_advertiser_summary, compare_advertisers, and more.

Available Tools

list_advertisers Lists all advertisers stored in AdIntel.
ParameterTypeDefaultDescription
No parameters
Returns: Array of advertiser objects with name, category, countries, and Sensor Tower platform IDs.
get_advertiser_summary Full latest Sensor Tower snapshot for one advertiser.
ParameterTypeDefaultDescription
advertiser_namestringrequiredExact advertiser name from the catalog
countrystringnullISO country code to filter results (e.g. "US", "BR")
Returns: Downloads, DAU, retention, impression share, rankings, demographics, reviews, review texts, creatives, and ASO keywords — all at latest available date.
compare_advertisers Latest single-metric snapshot across two or more advertisers side by side.
ParameterTypeDefaultDescription
advertiser_namesstringrequiredComma-separated names — e.g. "Binance, Coinbase, Kraken"
metricstring"downloads"One of: downloads, usage, retention, impression_share, rankings, reviews
countrystring"US"ISO country code
Returns: Latest metric values per advertiser with date.
get_metric_timeseries Daily historical data for one advertiser and metric — up to 90 days.
ParameterTypeDefaultDescription
advertiser_namestringrequiredExact advertiser name
metricstringrequiredOne of: downloads, usage, retention, impression_share, rankings, reviews
countrystring"US"ISO country code
start_datestringnullStart of range — YYYY-MM-DD
end_datestringnullEnd of range — YYYY-MM-DD
limitinteger90Max rows to return (max 90)
Returns: Chronological array of daily data points with date, country, and metric values.
get_collection_health Data freshness, last success, consecutive failures, and recent errors.
ParameterTypeDefaultDescription
advertiser_namestringnullSpecific advertiser, or omit to check all
Returns: Health record per advertiser with hours_since_success, consecutive_failures, and last error message.
get_collection_alerts Active alerts — stale data, repeated failures, never-collected advertisers.
ParameterTypeDefaultDescription
stale_hoursnumber48Hours threshold to flag data as stale
max_consecutive_failuresinteger3Failure count threshold to trigger alert
Returns: List of active alerts with severity (critical / warning) and type.
get_recent_collection_runs Recent collection job history with status, timing, and per-metric record counts.
ParameterTypeDefaultDescription
advertiser_namestringnullFilter by advertiser, or omit for all
platformstringnullFilter by platform (e.g. "sensortower")
limitinteger20Number of runs to return (max 100)
Returns: Run records with status, started_at, finished_at, message, and metadata including records written per metric.
request_advertiser Logs a missing advertiser for the team onboarding queue.
ParameterTypeDefaultDescription
namestringrequiredAdvertiser name to request
requested_bystringnullYour name or identifier
contextstringnullWhy you need this advertiser
Returns: Confirmation the request was logged with its status.
list_requested_advertisers Lists advertisers requested but not yet onboarded.
ParameterTypeDefaultDescription
No parameters
Returns: Pending requests with name, requester, context, status, and created date.
read_schema_text Returns the canonical SQL schema for the AdIntel database.
ParameterTypeDefaultDescription
No parameters
Returns: Raw SQL schema text — useful for understanding table structures or writing custom queries.

Appendix A: Data Availability

AdIntel exposes the following Sensor Tower data domains through its MCP tools.

Domain Key Metrics Dimensions
Downloads downloads, revenue date, country, OS (iOS / Android / unified)
Usage avg_dau, time_spent_min, sessions_per_day date, country
Retention d1, d3, d7, d14, d30, d60 cohort date, country
Impression Share sov_pct date, ad network, country
Demographics male_pct, female_pct age bracket, country
Rankings rank, is_featured date, category, chart type, country
Reviews avg_rating, rating_count, star distribution date, country
Review Texts body, sentiment, tags review date, country
Creatives creative_type, network, thumbnail_url first seen date
ASO Keywords rank, traffic_score, opportunity_score country, device

Tool coverage: get_metric_timeseries and compare_advertisers support: downloads, usage, retention, impression_share, rankings, reviews.
get_advertiser_summary surfaces all domains for one advertiser at once.
Most advertisers tracked in US only. Binance is also tracked in TR, BR, NG.
Coverage varies by advertiser — check get_collection_health if results look sparse.

Notes for Teammates

  • Use advertiser names exactly as they appear in the catalog — matching is case-sensitive
  • If an advertiser is missing, use request_advertiser to keep the onboarding queue visible to the whole team
  • If results look empty or sparse, check get_collection_health before assuming there's no data — the advertiser may have a recent collection failure
  • The MCP is read-only — it reflects what is already in the shared database
